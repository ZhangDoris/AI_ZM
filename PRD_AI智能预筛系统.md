## 1. 产品概述

### 1.1 产品定位

AI智能预筛系统是一款面向临床试验中心的智能化患者筛选平台。系统利用AI大模型能力，自动匹配患者病历数据与临床试验的入排标准，辅助研究者快速判断患者是否符合入组条件，大幅提升预筛效率。

### 1.2 核心价值

- **效率提升**: AI自动完成入排标准匹配，将单例患者审批从数十分钟缩短至分钟级
- **质量保障**: 标准化审批流程，降低人为遗漏风险
- **全链路追踪**: 从提交到最终结论，每个审批节点可追溯
- **多中心协作**: 支持多机构、多项目的统一管理

### 1.3 产品边界

- 面向临床试验中心（医院/研究机构），不面向患者端
- AI匹配结果为辅助参考，最终审批决策由研究者人工确认
- 支持预筛阶段和入组阶段的审批

## 2. 目标用户

| 角色 | 描述 | 核心场景 |
|------|------|----------|
| **CRC/研究助理** | 负责患者资料录入和初始筛选 | 提交筛选申请、查看AI匹配结果、填写预筛表 |
| **预筛审批员** | 负责预筛环节审批 | 审批AI匹配结果、修正错误判断、给出预筛结论 |
| **入组审批员** | 负责最终入组决策 | 复核预筛结论、确认入排条件、分配入组编号 |
| **项目管理员** | 负责项目和人员管理 | 管理项目信息、邀请码、脱敏配置、权限分配 |
| **数据统计人员** | 负责数据分析和报表 | 查看筛选统计、导出数据、生成报告 |

### 5.4 智能筛选 - 患者列表 · 匹配详情（思考链）

> **模块目标**：在智能筛选 → 患者列表 → 点击「匹配详情」时，展示与匹配详情并行计算的 AI 思考链结果。思考链与匹配详情解耦，异步加载，按思考状态展示不同界面。

#### 5.4.1 页面入口与触发时机

**入口路径**：
```
智能筛选 → 选中项目（如 SHR-2019-nCoV-001）→ 患者列表 → 点击行内「匹配详情」链接
```

**触发动作**：
- 用户点击「匹配详情」按钮 → **同时并行触发**两个后端任务：
  - 任务 A：拉取匹配详情（22 条入排标准比对）
  - 任务 B：触发该项目配置的思考链生成
- 弹窗立即打开，默认显示「匹配详情」Tab 内容
- 「思考链」Tab 按思考状态自适应呈现三种界面

**异步加载策略**：
| 任务 | 加载策略 | 用户感知 |
|------|----------|----------|
| 匹配详情 | 同步优先 | 弹窗打开即展示 |
| 思考链 | 异步后台 | 按思考状态分情形渲染（见 5.4.4） |
| 通知机制 | 思考完成时 | 站内消息 / 邮件通知（不强制弹窗） |

#### 5.4.2 思考链 Tab 显隐规则

| 条件 | 是否显示 Tab | 说明 |
|------|--------------|------|
| 项目已配置 ≥ 1 条思考链 | ✅ 显示 | 正常入口 |
| 项目未配置思考链 | ❌ 隐藏 | 整个 Tab 不渲染 |

**判定方式**：进入详情页时，根据 `projectId` 查询项目级思考链配置；配置为空则不渲染该 Tab。

#### 5.4.3 思考链 Tab 公共元素

**类型选择器**（顶部左侧）：
- 按钮组：入排条件 / 筛选表 / 自定义
- 当前选中态：边框 + 文字变蓝（#1677ff）
- **点击效果**：切换类型时，列表区域重新拉取该类型（inclusion / screening / custom）的思考链结果并刷新

**「全部思考」按钮**（顶部右侧）：
- 主色按钮（蓝底白字），图标 🧠
- **显隐规则（★ 与原型一致）**：仅在该患者思考链处于「思考失败」状态时显示；「思考中」「思考成功（含异步自动完成）」状态下**不显示**该按钮，避免无效重试
- **点击效果**：触发后端对该患者所有思考链重新执行 → 列表区域切换为「思考中」占位 → 完成后（演示为 3 秒）转为成功并重渲染；同时弹出 Toast：「🧠 AI 正在分析处理中，请稍后…思考成功之后，将会通知您」

**列表区域（每条思考链数据）**：
- 每条思考链为独立可折叠卡片，含：序号徽标（蓝底圆标）、名称、「AI思考」按钮、「查看/编辑思考链」按钮、展开/收起箭头
- 默认展开显示该条完整步骤内容（Markdown 格式，带步骤编号与说明）
- 状态标签：已完成（绿）/ 思考中（紫）/ 思考失败（红）

**点击每条思考链数据的效果（核心交互）**：
1. **点击卡片标题栏（序号 / 名称 / 箭头区域，按钮除外）** → 展开 / 收起该条思考链的步骤内容；箭头图标在 `▴ / ▾` 间同步切换；「查看/编辑」按钮已阻止事件冒泡，不会误触发折叠
2. **点击「AI思考」按钮（单条）** → 仅对该条思考链重新执行：该卡片所在列表区切换为「思考中」→ 完成后（演示 3 秒）转回成功并刷新；弹出 Toast 提示（权限见 5.4.5）
3. **点击「查看/编辑思考链」按钮** → 打开思考链查看/编辑弹窗：
   - 医学经理 / 项目经理 / 管理员：可编辑步骤逻辑
   - 其他角色（CRC、预筛 / 入组审批员）：只读查看
4. **点击展开箭头图标** → 同第 1 条，仅用于视觉指示当前展开 / 收起状态

**底部免责声明**（固定底栏）：
- 黄色横幅：以上内容由 AI 生成，仅供参考，请谨慎鉴别。如果需要校正思考链，可以修改思考链的逻辑。

#### 5.4.4 思考链三种状态界面（核心）

##### 状态 ① 思考成功（已匹配 + 思考完成）

**触发条件**：`matchStatus = '已匹配'` 且 `thinkChainStatus = '完成'`

**界面表现**：
- 完整展示思考链列表卡片
- 每条链显示「已完成」绿色标签
- 步骤内容正常渲染，可展开/收起（点击效果见 5.4.3）
- 「全部思考」按钮**不显示**（思考成功态按 5.4.3 显隐规则隐藏）
- 「AI思考」按钮在每条链上独立可用（点击重新思考该条）

**参考数据样例**（王*明 / WYNA-CN001）：
- 思考链 1：ASCVD心血管风险评估（3 步）
- 思考链 2：肾功能评估（eGFR/UACR）（2 步）
- 用时：2 分 38 秒

##### 状态 ② 匹配中（思考链尚未生成）

**触发条件**：`matchStatus = '匹配中'`（或 `待匹配`）

**界面表现**：
- 类型选择器正常显示
- 「全部思考」按钮**不显示**（思考中态按 5.4.3 显隐规则隐藏，避免重复触发）
- 列表区域被 loading 占位卡片替换，展示：
  - 紫罗兰色脉冲 🧠 脑图标
  - 标题：「**正在思考中，请稍后再来**」
  - 描述：告知患者姓名/筛选号、并行任务机制、完成后通知方式
  - 实时信息：已等待时长、队列进度
  - 操作按钮：「思考完成时通知我」「返回列表」

**交互约束**：
- 弹窗其他 Tab（匹配详情/预筛表/病史）正常可用
- 不阻塞用户继续查看其他内容

##### 状态 ③ 思考失败（已匹配但思考链生成失败）

**触发条件**：`matchStatus = '已匹配'` 且 `thinkChainStatus = '失败'`

**界面表现**：
- 类型选择器正常显示
- 「全部思考」按钮**高亮显示**（核心重试入口，点击触发该患者全部思考链重新执行）
- 列表区域被失败占位卡片替换，展示：
  - 红色 ⭕ 错误图标
  - 标题：「**思考失败，请重新思考**」
  - 描述：失败原因（错误码/异常信息/超时说明）
  - 红色错误详情卡片：失败时间、错误码、已重试次数、建议操作
  - 操作按钮：「全部思考」「联系管理员」「返回列表」

**常见失败原因**：
| 错误码 | 含义 | 建议处理 |
|--------|------|----------|
| `TC_GEN_FAILED_504` | RAG 检索服务超时 | 30s 后重试 |
| `TC_GEN_FAILED_500` | LLM 服务异常 | 1min 后重试 |
| `TC_DATA_MISSING` | 患者病历数据缺失 | 联系 CRC 补充 |
| `TC_CONFIG_ERROR` | 思考链配置错误 | 联系医学经理/项目经理 |

| children | Array | 子操作行列表（可选，有子操作时显示展开按钮） |

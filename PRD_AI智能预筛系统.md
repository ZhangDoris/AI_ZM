## 1. 产品概述

### 1.1 产品定位

AI智能预筛系统是一款面向临床试验中心的智能化患者筛选平台。系统利用AI大模型能力，自动匹配患者病历数据与临床试验的入排标准，辅助研究者快速判断患者是否符合入组条件，大幅提升预筛效率。

### 1.2 核心价值

- **效率提升**: AI自动完成入排标准匹配，将单例患者审核从数十分钟缩短至分钟级
- **质量保障**: 标准化审核流程，降低人为遗漏风险
- **全链路追踪**: 从提交到最终结论，每个审核节点可追溯
- **多中心协作**: 支持多机构、多项目的统一管理

### 1.3 产品边界

- 面向临床试验中心（医院/研究机构），不面向患者端
- AI匹配结果为辅助参考，最终审核决策由研究者人工确认
- 支持预筛阶段和入组阶段的审核

## 2. 目标用户

| 角色 | 描述 | 核心场景 |
|------|------|----------|
| **CRC/研究助理** | 负责患者资料录入和初始筛选 | 提交筛选申请、查看AI匹配结果、填写预筛表 |
| **预筛审核员** | 负责预筛环节审核 | 审核AI匹配结果、修正错误判断、给出预筛结论 |
| **入组审核员** | 负责最终入组决策 | 复核预筛结论、确认入排条件、分配入组编号 |
| **项目管理员** | 负责项目和人员管理 | 管理项目信息、邀请码、脱敏配置、权限分配 |
| **数据统计人员** | 负责数据分析和报表 | 查看筛选统计、导出数据、生成报告 |

**匹配详情表格**:

**位置**: 详情页右侧，Tab内容区旁边的固定侧栏（宽度300px）

**功能**: 展示当前患者最近3条审核记录的精简摘要

**精简摘要展示**:

| 元素 | 说明 |
|------|------|
| 彩色圆点 | 绿色=入组，红色=排除，橙色=待定 |
| 摘要信息 | 审核人 · 审核结论 + 意见摘要（前30字） |

| duration | String | 操作耗时 |

**行数据结构**: 父行包含 `children` 数组（子操作行列表），仅当有子操作时显示 ▶ 展开按钮。

**原型数据模拟**:
- id=2: 有入组审核记录（1条父行 + 1条子行）
- id=7: 完整的预筛+入组审核记录（各1条父行 + 1条子行 + 批注）
- id=10: 完整的预筛+入组审核记录 + 批注
- id=8: 仅有入组审核记录
- id=1: 仅有预筛提交记录

## 6. 数据模型

### 6.1 审核记录 (auditData)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Number | 唯一标识 |
| screenNo | String | 筛选号 |
| org | String | 所属机构 (hengrui/kexin/yunying) |
| auditType | String | 审核类型 (预筛审核/入组审核) |
| patientNo | String | 患者编号 |
| gender | String | 性别 (男/女) |
| age | Number | 年龄 |
| planNo | String | 方案编号 |
| projectName | String | 项目名称 |
| centerNo | String | 中心号 |
| center | String | 中心名称 |
| matchRate | String | AI匹配率 (百分比) |
| submitter | String | 提交人 |
| status | String | 审核状态 (pending/done) |
| reviewStage | String | **审核环节** (待分配/预筛审核中/入组审核中/已完成（入组）/已完成（排除）) |

### 6.2 审核时间线 - 右侧栏摘要 (auditTimelineData)

| 字段 | 类型 | 说明 |
|------|------|------|
| reviewer | String | 审核人姓名 |
| role | String | 审核角色 (预筛审核员/入组审核员) |
| opinion | String | 审核意见（完整文本，侧栏截取前30字） |
| time | String | 审核时间 (YYYY-MM-DD HH:mm:ss) |
| result | String | 审核结论 (建议入组/建议排除/确认入组/确认排除) |

**数据结构**: `{ [recordId]: TimelineEntry[] }`，以审核记录ID为Key的Map结构。当某条记录无审核历史时，该Key不存在。

### 6.3 审核流程 (auditFlowData) ★ 新增

用于审核记录弹窗中的分组表格，按阶段（预筛/入组）组织。

| 字段 | 类型 | 说明 |
|------|------|------|
| operator | String | 操作人姓名 |
| role | String | 角色（CRC / CRA专员 / 预筛审核员 / 入组审核员） |
| statusClass | String | 状态类型 (submitted=已提交 / approved=通过 / rejected=拒绝 / pending=待审核) |
| time | String | 操作时间 |
| duration | String | 操作耗时 |
| children | Array | 子操作行列表（可选，有子操作时显示展开按钮） |

**数据结构**: `{ [recordId]: { enrolling: FlowRow[], prescreen: FlowRow[] } }`

**交互规则**:
- 父行（如CRC提交）默认收起其 children（如CRA审核结果）
- 点击父行 ▶ 按钮展开/收起子行
- 子行背景色为 #f8f9fb 区分父行

# AI改进分析报告页 - PPRD文档

## 1. 文档概述

### 1.1 基本信息
| 字段 | 内容 |
|------|------|
| **文档名称** | AI改进分析报告页产品需求文档(PPRD) |
| **版本号** | v1.0 |
| **创建日期** | 2026-06-16 |
| **最后更新** | 2026-06-16 |
| **负责人** | 产品团队 |
| **关联页面** | ai-improve-report.html、ai-improve-report-detail.html |

### 1.2 产品背景
AI改进分析功能是智能预筛系统中的核心模块，用于对患者数据进行智能化分析和改进建议生成。设置页面为管理员和技术人员提供系统的配置、调度和手动执行能力，确保AI分析的灵活性和可控性。

### 1.3 目标用户
- **系统管理员**：负责系统配置、定时任务管理和权限控制
- **项目经理**：监控分析执行情况，查看运行统计
- **临床研究人员**：触发特定项目的AI分析，查看结果报告
- **技术支持**：排查执行问题，优化执行策略

## 2. 功能需求

### 2.1 搜索功能

#### 功能描述
支持按项目编号、方案编号、项目名称进行组合搜索，快速定位目标项目。

#### 详细规则
| 元素 | 规格说明 |
|------|----------|
| **位置** | 列表区域上方，工具栏右侧 |
| **控件类型** | 搜索输入框 + 搜索按钮 |
| **占位符** | "请输入项目编号/方案编号/项目名称搜索..." |
| **快捷键** | Enter键触发表单提交 |
| **宽度** | 最小300px，自适应扩展 |

#### 交互行为
- 实时搜索，输入≥2个字符时开始匹配
- 搜索范围：项目编号、方案编号、项目名称三者任意匹配
- 搜索结果即时刷新列表
- 清除按钮清空搜索条件

---

### 2.2 列表展示

#### 功能描述
列表包含以下列：项目编号、方案编号、项目名称、病例数、AI准确率、通过率、问题数、建议数、状态。支持点击表头排序。

#### 详细规则

##### 2.2.1 列表字段规格
| 字段名 | 类型 | 对齐 | 说明 |
|--------|------|------|------|
| **项目编号** | 字符串 | 左对齐 | 可点击跳转到详情页 |
| **方案编号** | 字符串 | 左对齐 | - |
| **项目名称** | 字符串 | 左对齐 | - |
| **病例数** | 数字 | 右对齐 | 千分位格式化 |
| **AI准确率** | 百分比 | 右对齐 | 蓝色高亮 |
| **通过率** | 百分比 | 右对齐 | - |
| **问题数** | 数字 | 右对齐 | 红色标注>10的数据 |
| **建议数** | 数字 | 右对齐 | - |
| **状态** | 状态标签 | 居中 | 已完成/分析中/待处理 |

##### 2.2.2 排序规则
| 字段 | 升序图标 | 降序图标 | 默认排序 |
|------|----------|----------|----------|
| 项目编号 | ↑ | ↓ | 降序 |
| AI准确率 | ↑ | ↓ | 降序 |
| 通过率 | ↑ | ↓ | 降序 |
| 病例数 | ↑ | ↓ | 降序 |

##### 2.2.3 分页
| 元素 | 规格说明 |
|------|----------|
| **每页条数** | 10/20/50条可选 |
| **当前页** | 高亮显示 |
| **总页数** | 显示"第 X/Y 页" |

#### 交互行为
- 点击项目编号列跳转到详情页
- 点击表头切换排序方向
- 鼠标悬停行显示高亮背景（#f5f5f5）
- 空列表展示占位图和"暂无数据"提示

---

### 2.3 报告详情 - 总体概况

#### 功能描述
点击查看项目具体分析，展示项目基本信息（编号、名称、病例数）、AI准确率、通过率、问题数、建议数，以及关键指标完成情况（进度条显示）。

#### 详细规则

##### 2.3.1 入口方式
| 交互 | 触发条件 | 动作 |
|------|----------|------|
| **点击列表行** | 鼠标点击列表中的任一项目行 | 打开详情页（新窗口/覆盖层） |
| **点击项目编号** | 鼠标点击项目编号单元格 | 打开详情页（新窗口/覆盖层） |

##### 2.3.2 页面头部信息
| 元素 | 规格说明 |
|------|----------|
| **项目标题** | 20px，加粗，深黑色，显示项目编号+名称 |
| **分析日期** | 14px，灰色，格式：YYYY-MM-DD HH:mm |
| **病例总数** | 14px，灰色，显示格式：共 X 例病例 |
| **步骤条** | 两步，圆点编号+标题+描述，点击切换 |

##### 2.3.3 核心指标统计卡
| 字段 | 样式 | 数据源 |
|------|------|--------|
| **项目编号** | 20px，蓝色 | 项目基本信息 |
| **方案编号** | 18px，蓝色 | 项目基本信息 |
| **病例数** | 32px，蓝色 | 项目数据 |
| **AI准确率** | 32px，蓝色 | AI分析结果 |

#### 交互行为
- 点击列表行后，新页面以 `window.open()` 方式打开
- 页面URL携带项目ID参数：`?projId=XXX`
- 详情页顶部"返回"按钮使用 `window.history.back()` 返回

### 2.4 具体问题与建议

#### 功能描述
逐一列出系统发现的问题，每个问题配改进建议。例如"他汀类药物判定不太准，建议加上XX条件"。

#### 详细规则

##### 2.4.1 改进建议卡片
| 元素 | 规格说明 |
|------|----------|
| **优先级徽章** | P0红色(#ff4d4f)、P1琥珀色(#faad16)、P2蓝色(#1677ff) |
| **建议编号** | 14px，加粗，深黑色，格式：建议 #X |
| **影响数量** | 12px，灰色，显示格式：影响 X 例 (Y%) |
| **问题描述** | 13px，琥珀色，显示具体问题 |
| **改进方案** | 文本域，60px最小高度，13px字号 |
| **提示词** | 等宽字体文本域，60px最小高度，背景浅蓝色 |

##### 2.4.2 标准关联说明
| 元素 | 规格说明 |
|------|----------|
| **数据来源标注** | 11px，灰色背景备注框，左侧蓝色边框 |
| **内容格式** | "此建议对应的标准为 **[入选标准/排除标准XX]** ，与入排标准数据直接关联" |
| **图标** | Font Awesome数据库图标，蓝色 |

##### 2.4.3 启用开关
| 元素 | 规格说明 |
|------|----------|
| **预筛启用开关** | 拨动开关，宽度44px，高度24px |
| **入排启用开关** | 拨动开关，宽度44px，高度24px |
| **开启状态** | 蓝色渐变背景，右侧滑块，阴影发光 |
| **关闭状态** | 灰色背景，左侧滑块 |
| **标签文字** | "预筛"/"入排"（灰色），"已启用"（蓝色加粗）/"已禁用"（灰色） |

###### 开关交互
- 点击开关触发状态切换，切换动画0.3s
- 预筛启用：表示此建议在预筛阶段生效
- 入排启用：表示此建议在入排阶段生效
- 开关状态变更时实时保存到本地缓存

##### 2.4.4 备注说明
| 元素 | 规格说明 |
|------|----------|
| **提示词备注** | 11px，浅灰背景框，左侧绿色边框 |
| **内容** | "备注说明：预筛启用表示此建议在预筛阶段生效，入排启用表示此建议在入排阶段生效" |
| **图标** | Font Awesome灯泡图标，绿色 |

#### 交互行为
- 改进方案和提示词语义可直接编辑，修改后立即反映在UI中
- 开关状态变更时提供视觉反馈（颜色过渡动画）
- 保存按钮点击后弹出确认提示

### 2.5 AI判断 vs 人工判断 对比

#### 功能描述
展示AI判断与人工判断的一致性分析，包括通过率对比和判断分类统计。

#### 详细规则

##### 2.5.1 通过率对比
| 元素 | 规格说明 |
|------|----------|
| **对比方式** | 双进度条并列显示 |
| **AI通过率** | 蓝色进度条，右侧显示百分比数值 |
| **人工通过率** | 绿色进度条，右侧显示百分比数值 |
| **标签** | 12px灰色标签显示"AI判断通过率"/"人工判断通过率" |

##### 2.5.2 判断分类矩阵
| 类别 | 颜色 | 显示格式 |
|------|------|----------|
| **双方一致通过** | 绿色(#52c41a) | X 例 (Y%) |
| **双方一致拒绝** | 蓝色(#1677ff) | X 例 (Y%) |
| **AI拒绝，人工通过** | 红色(#ff4d4f) | X 例 (Y%) |
| **人工拒绝，AI通过** | 琥珀色(#fa8c16) | X 例 (Y%) |

每个分类展示为带左侧彩色边框的卡片：
| 元素 | 规格说明 |
|------|----------|
| **卡片边框** | 左侧4px彩色条，标识类别 |
| **类别名称** | 13px，加粗，同色系颜色 |
| **数量统计** | 13px，加粗，同色系颜色 |
| **辅助说明** | 11px，灰色，说明含义（如"AI过于严格，误拒可通过病例"） |

### 2.7 保存功能

#### 功能描述
保存用户对改进方案的修改，支持批量保存所有更改。

#### 详细规则
| 元素 | 规格说明 |
|------|----------|
| **按钮位置** | 详情页底部，右侧对齐 |
| **按钮文案** | "保存改进方案" |
| **按钮样式** | 蓝色渐变背景，白色文字，10px 24px padding |
| **备注说明** | "保存的改进方案将用于下一次AI提示词更新"（12px，灰色，右侧显示） |
| **成功提示** | toast消息："改进方案已保存！修改后将用于下一次AI提示词。" |

#### 交互行为
1. 用户修改改进方案或提示词
2. 点击"保存改进方案"按钮
3. 按钮短暂显示"保存中..."状态
4. 保存成功后弹出toast提示
5. 若保存失败，显示错误提示并保留修改内容

## 6. 交互细节

### 6.1 动效设计

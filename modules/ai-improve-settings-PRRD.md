# AI改进分析设置页 - PPRD文档

## 1. 文档概述

### 1.1 基本信息
| 字段 | 内容 |
|------|------|
| **文档名称** | AI改进分析设置页产品需求文档(PPRD) |
| **版本号** | v1.0 |
| **创建日期** | 2026-06-16 |
| **最后更新** | 2026-06-16 |
| **负责人** | 产品团队 |
| **关联页面** | ai-improve-settings.html |

### 1.2 产品背景
AI改进分析功能是智能预筛系统中的核心模块，用于对患者数据进行智能化分析和改进建议生成。设置页面为管理员和技术人员提供系统的配置、调度和手动执行能力，确保AI分析的灵活性和可控性。

### 1.3 目标用户
- **系统管理员**：负责系统配置、定时任务管理和权限控制
- **项目经理**：监控分析执行情况，查看运行统计
- **临床研究人员**：触发特定项目的AI分析，查看结果报告
- **技术支持**：排查执行问题，优化执行策略

## 2. 功能需求

### 2.1 前置条件提示模块

#### 功能描述
在页面顶部展示AI改进分析执行的前置条件，帮助用户了解系统要求。

#### 详细规则
| 元素 | 规格说明 |
|------|----------|
| **位置** | 页面顶部，导航栏下方 |
| **背景色** | 渐变橙黄色（#fff7e6 → #ffe7ba）|
| **边框** | 1px 橙色边框（#ffd591）|
| **圆角** | 12px |
| **图标** | 警告图标，橙色（#fa8c16），20px |
| **标题** | "该功能依赖以下前置条件"，14px，加粗，深橙色 |
| **列表项** | 13px，行高1.8，带项目符号 |

#### 前置条件清单
1. ✅ 项目已完成伦理审批，患者数据已完整录入系统
2. ✅ 目标项目状态为"进行中"，至少有一条「拒绝」、「已排除」或「已有批注」记录
3. ✅ 已配置AI分析模型及API密钥
4. ⚠️ 定时任务需在工作日（周一至周五）配置

#### 交互行为
- 静态展示，无交互
- 条件满足时正常显示，不满足时标题变为红色警告样式

### 2.2 项目信息展示模块

#### 功能描述
展示当前AI改进分析所依赖的项目基础信息和患者数据分布。

#### 详细规则

##### 2.2.1 AI字段信息卡片
| 元素 | 规格说明 |
|------|----------|
| **布局** | 网格布局，每行3列 |
| **卡片样式** | 灰色背景（#fafafa），圆角8px |
| **必填字段** | 蓝色左边框（3px #1677ff），右上角显示蓝色徽章"必填" |
| **可选字段** | 灰色左边框（3px #d9d9d9），半透明（opacity: 0.85）|
| **悬浮效果** | 背景变蓝（#f0f5ff），上移1px，阴影效果 |

##### 2.2.2 患者数据分布
| 字段 | 说明 |
|------|------|
| **总患者数** | 当前项目下所有患者记录总数 |
| **待分析患者数** | 状态为「拒绝」、「已排除」或「已有批注」的患者数 |
| **占比** | 待分析患者数 / 总患者数 × 100% |

#### 交互行为
- 点击AI字段卡片可查看详情（未来扩展）
- 数据自动刷新，无需手动操作

### 2.3 定时执行配置模块

#### 功能描述
允许管理员配置AI改进分析的定时执行策略，实现自动化数据分析。

#### 详细规则

##### 2.3.1 定时开关
| 元素 | 规格说明 |
|------|----------|
| **控件类型** | 自定义Switch Toggle |
| **开启状态** | 蓝色渐变（#1677ff → #0958d9），右侧滑块，阴影发光效果 |
| **关闭状态** | 灰色（#d9d9d9），左侧滑块 |
| **标签文字** | "已启用" / "已禁用" |
| **切换动画** | 0.3s 缓动 |

##### 2.3.2 执行周期配置
| 字段 | 规格说明 |
|------|----------|
| **周期选择** | 固定"每周" |
| **星期选择** | 下拉框，选项：选择/一/二/三/四/五/六/日 |
| **时间选择** | 标准时间输入框，默认值 22:00 |
| **提示文本** | "自动执行"（13px，灰色）|

##### 2.3.3 执行策略说明
- **文本内容**："遍历所有项目，分析有「拒绝」、「已排除」或「已有批注」的患者数据"
- **样式**：13px，灰色（#555）
- **交互**：静态展示，无交互

##### 2.3.4 执行时间信息
| 字段 | 规格说明 |
|------|----------|
| **上次执行** | 格式：YYYY-MM-DD HH:mm，灰色（#333）|
| **下次执行** | 格式：YYYY-MM-DD HH:mm，蓝色高亮（#1677ff），加粗（500）|
| **禁用状态** | 下次执行显示"未设置"，灰色（#888）|

#### 交互行为
- 开关切换时实时保存配置
- 星期和时间变更时自动重新计算下次执行时间
- 下次执行时间实时更新显示

#### 异常处理
- 如果当前时间已超过22:00，下次执行时间自动调整为下周同日
- API调用失败时显示错误提示，保留原有配置

### 2.4 手动执行模块

#### 功能描述
允许用户手动触发AI改进分析，支持单项目和批量项目两种模式。

#### 详细规则

##### 2.4.1 项目选择器
| 元素 | 规格说明 |
|------|----------|
| **控件类型** | 下拉选择框 |
| **占位符** | "选择项目..." |
| **选项数量** | 1（全部项目）+ N（实际项目数）|
| **全部项目选项** | "📦 全部项目（5个）"|
| **项目选项** | 项目ID + 项目全称 + 研究阶段 |
| **宽度** | 自适应，最小200px |

##### 2.4.2 执行按钮
| 元素 | 规格说明 |
|------|----------|
| **文案** | "▶ 执行分析" |
| **样式** | 蓝色渐变背景，白色文字，圆角6px |
| **尺寸** | 小按钮（padding: 6px 12px，font-size: 12px）|
| **禁用状态** | 灰色背景（#d9d9d9），cursor: not-allowed |

##### 2.4.3 执行状态指示器
| 状态 | 样式 |
|------|------|
| **空闲** | 灰色圆点 + "空闲"文字，灰色背景 |
| **分析中** | 黄色脉冲圆点 + "分析中..."文字，浅黄背景（#fff7e6）|
| **已完成** | 绿色圆点 + "已完成"文字，浅绿背景（#f6ffed）|

#### 交互行为

##### 执行流程
1. **初始状态**：状态指示器显示"空闲"，按钮可点击
2. **验证阶段**：
   - 未选择项目 → 弹出警告提示"请先选择要分析的项目！"
   - 已选择项目 → 进入执行阶段
3. **执行阶段**：
   - 按钮禁用，文案变为"⏳ 执行中..."
   - 状态指示器变为"分析中..."（黄色脉冲动画）
   - 执行耗时：约3秒（实际业务中取决于数据量）
4. **完成阶段**：
   - 按钮恢复，文案回到"▶ 执行分析"
   - 状态指示器变为"已完成"（绿色）
   - 弹出执行结果模态框

##### 模态框内容
| 字段 | 说明 |
|------|------|
| **标题** | "AI改进分析执行成功" |
| **分析项目数** | 单项目显示1，全部项目显示实际数量 |
| **涉及病例数** | 显示本次分析覆盖的患者数量 |
| **按钮** | "取消"（次要）、"查看报告"（主要，跳转至报告页）|

#### 异常处理
- 执行过程中API超时 → 状态变为"失败"，按钮恢复可点击
- 服务器异常 → 弹出错误提示模态框，显示错误详情
- 用户权限不足 → 按钮禁用，tooltip提示"无执行权限"

## 3. 非功能需求

### 3.1 性能要求
| 指标 | 要求 |
|------|------|
| **页面加载时间** | ≤ 2秒 |
| **配置保存延迟** | ≤ 500ms |
| **手动分析响应** | 首屏响应 ≤ 1秒 |
| **状态指示器刷新** | ≤ 200ms |

### 3.2 兼容性要求
| 维度 | 支持范围 |
|------|----------|
| **浏览器** | Chrome 90+, Edge 90+, Firefox 88+, Safari 14+ |
| **屏幕分辨率** | 最小1280×768，推荐1920×1080 |
| **移动端** | 适配768px以下屏幕（响应式布局）|

### 3.3 安全要求
| 要求 | 说明 |
|------|------|
| **权限控制** | 定时配置仅管理员可修改 |
| **操作审计** | 所有配置变更和执行操作记录日志 |
| **数据脱敏** | 患者数据展示符合隐私保护规范 |

### 3.4 可用性要求
| 要求 | 说明 |
|------|------|
| **错误提示** | 所有错误均有明确的中文提示 |
| **操作反馈** | 每次操作都有视觉反馈（颜色变化、动画、toast）|
| **无障碍** | 支持键盘导航，关键控件有aria-label |

## 5. 数据接口

### 5.1 获取设置信息
```http
GET /api/ai-improve/settings
```
**响应示例**：
```json
{
  "scheduleEnabled": true,
  "scheduleDay": 5,
  "scheduleTime": "22:00",
  "lastRun": "2026-06-05 22:00",
  "nextRun": "2026-06-12 22:00",
  "fields": [
    {"key": "patient_id", "name": "患者ID", "required": true},
    {"key": "status", "name": "状态", "required": true},
    {"key": "notes", "name": "批注", "required": false}
  ],
  "stats": {
    "totalPatients": 2988,
    "pendingAnalysis": 988,
    "percentage": 33
  }
}
```

### 5.2 更新定时配置
```http
PUT /api/ai-improve/settings/schedule
Content-Type: application/json

{
  "enabled": true,
  "day": 5,
  "time": "22:00"
}
```

### 5.3 手动触发分析
```http
POST /api/ai-improve/execute
Content-Type: application/json

{
  "projectId": "__ALL__",
  "projectIds": ["SHR-1918-302", "CRD-2024-HX"]
}
```
**响应**：
```json
{
  "status": "success",
  "projectCount": 3,
  "caseCount": 2988,
  "reportUrl": "/ai-improve-report.html"
}
```

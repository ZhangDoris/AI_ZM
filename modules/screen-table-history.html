匹配管理 - 筛选表日志（历史）

📋 PRD 需求说明

页面定位

  • 模块：AI 智能预筛系统 - 匹配管理
  • 功能：显示所有筛选表数据的完整历史记录
  • 用户：管理员、医学经理（查看历史筛选记录）

页面组成

  • 搜索区：多维度筛选查询
  • 列表区：所有患者的全部筛选记录（不过滤）
  • 查看数据页：完整筛选运行信息和逐条标准匹配详情

搜索区筛选项

  • 患者编号：文本输入，精确匹配
  • 匹配状态：下拉选择（全部/识别中/匹配成功/匹配失败）
  • 匹配人：文本输入，模糊匹配
  • 项目编码：文本输入，精确匹配
  • 方案编号：文本输入，模糊匹配
  • 是否删除：下拉选择（全部/是/否）
  • 时间范围：日期区间选择

列表展示字段

  • 序号、方案编号、项目编码、项目名称、患者姓名
  • 匹配状态（灰色/绿色/红色标签）
  • 操作时间、开始匹配时间、结束匹配时间
  • 操作：查看数据

业务规则

  • 列表展示所有患者的全部筛选记录，不过滤，同一条患者多条记录均显示
  • 筛选表历史日志不可编辑和删除，仅支持查看和导出

查看数据页面

  • 页面结构与"筛选表日志（最新）"的"查看数据"页面一致
  • 展示完整筛选运行信息和逐条标准匹配详情

数据输出

  • 筛选表日志（历史）列表（分页，支持每页10条/20条/50条切换）
  • 查看数据页面（完整匹配运行信息和逐条标准判定）
  • 日志导出

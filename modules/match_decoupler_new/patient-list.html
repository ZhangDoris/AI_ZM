修改摘要

注意：列表的项目匹配和 病史与资料的项目匹配思路完全一致，目前的效果 在列表上实现，病史与资料可参考该思路

  • 点击了项目匹配之后可以直接预提交审批，预提交审核的按钮。
  • 匹配中，未提交的，也不在预提交中的，可以预提交
  • 已经预提交的按钮，在序号上显示预提交状态，然后操作列有取消提交，提交成功，就不显示这个状态和按钮了， 只要系统还没有协助提交，就存在这俩状态，如果系统提交成功，就不显示了

注意：

  • 匹配成功后，系统自动提交，如果匹配失败了（注意三次容错机制），就需要通知再次点击项目匹配，才可以进行提交审核了,让把之前的预提交过程再做一遍
  • 取消预提交/匹配失败再设置预提交，不过内容都需要回显

📌 操作列按钮显示规则：

  • ① 预提交中（匹配中/已匹配）→ 显示「取消预提交」按钮，不显示提交审核
  • ② 匹配中 + 未预提交 → 显示「预提交」按钮，不显示提交审核
  • ③ 匹配完成 + 未预提交 → 显示「提交审核」按钮，不显示预提交
  • ④ 待匹配（未发起匹配）→ 显示「提交审核」按钮
  • ⑤ 预提交中状态标注 → 序号列显示「预提交中」红底标签（参考第3条数据效果）

交互流程

A. 通过项目匹配按钮：

  • 前置条件：患者已上传病史与资料
  • 第1条数据(本周) 点击提交审核，看到入排只有一个版本的效果
  • 第2条数据(本周)点击项目匹配，弹出「入排版本选择」弹窗（选择项目编码+版本）→ 点击确定 → 显示「系统匹配中」弹窗 → 点击预提交审批 → 与现有提交审核弹窗一致
  • 第3条数据(本周)状态为「匹配中」，无预提交 → 操作列显示「预提交」按钮 → 点击预提交 → 弹出提交审核弹窗 → 系统自动标记为「预提交中」并显示取消预提交
  • 第4条数据(下周) 点击项目匹配，弹出确认框：「检测到您的资料未发生变更。是否确定要基于当前资料再次进行项目匹配？」→ 点击项目匹配 → 显示「系统匹配中」弹窗
  • 第5条数据(下周)已提交审核，资料未变更 → 点击项目匹配 → 弹出重新匹配确认弹窗 → 点击开始匹配 → toast「正在匹配，请稍后」
  • 第6条数据(下周) 已提交审核，文件有变动 → 点击项目匹配 → 弹出项目匹配确认弹窗 → 点击开始匹配 → toast「正在匹配，请稍后」
  • 第7条数据(本周) 主知情审核/需要填写预筛号和知情时间的 → 点击项目匹配 → 「入排版本选择」弹窗（选择项目编码+版本）→ 点击确定 → 显示「系统匹配中」弹窗 → 点击预提交审批 → 与现有提交审核弹窗一致 → 点击提交审核，弹出编辑页面，需要填写患者知情时间和筛选号
  • 注意：对于点击项目匹配可以预提交 ， 所有可以提交的相关人员都可以操作，只要初始这条数据没有提交审核（包括预筛、主知情、入组等等）点击项目匹配，就可以选择 预提交，如果已经在 审核流程中了，点击项目匹配 就不需要 预提交了，直接匹配即可；取消预提交按钮的操作权限 是所有可以提交本次数据的相关人员

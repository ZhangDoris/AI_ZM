# PRD - 匹配管理模块

## 一、模块概述

### 1.1 功能定位

匹配管理记录和追踪患者筛选匹配过程的日志数据，为问题排查和效果分析提供依据。

### 1.2 目标用户

- **管理员**：查看所有匹配日志，进行系统维护和数据分析
- **医学经理**：查看筛选表日志，排查匹配问题，进行效果分析

### 1.3 模块组成

匹配管理包含三个子页面：

1. **匹配日志（最新）**：从项目角度出发，查询每个项目下患者的最新匹配记录
2. **筛选表日志（最新）**：从项目角度出发，查询每个项目下患者的最新筛选表数据
3. **筛选表日志（历史）**：显示所有筛选表数据的完整历史记录

---

## 二、页面一：匹配日志（最新）

### 2.1 功能描述

从项目角度出发，查询每个项目下患者的最新匹配记录，支持查看匹配详情和重新匹配。

### 2.2 业务规则

1. 列表按项目维度聚合，每个患者只显示最新的匹配记录（同一患者多条记录时只显示最新一条）
2. 匹配结果状态分为：识别中、匹配成功、匹配失败
3. 匹配成功状态用绿色标签显示，匹配失败用红色标签显示，识别中用灰色标签显示
4. 点击"匹配详情"可进入详情页面，查看完整的匹配运行信息
5. 点击"重新匹配"触发后端匹配接口调用，重新执行匹配逻辑
6. 重新匹配过程中按钮显示加载状态，防止重复提交
7. 匹配日志不可编辑和删除，仅支持查看和导出
8. 支持按患者编号、匹配状态、匹配人、项目编码、时间范围筛选
9. 是否超过10min：标记匹配耗时是否超过10分钟的异常情况

### 2.3 搜索区

| 筛选项 | 说明 |
|--------|------|
| 患者编号 | 文本输入，精确匹配 |
| 匹配状态 | 下拉选择：全部 / 识别中 / 匹配成功 / 匹配失败 |
| 匹配人 | 文本输入，模糊匹配 |
| 项目编码 | 文本输入，精确匹配 |
| 是否删除 | 下拉选择：全部 / 是 / 否 |
| 时间范围 | 日期区间选择（开始日期 - 结束日期） |
| 是否超过10min | 下拉选择：全部 / 是 / 否 |

**操作按钮**：查询、重置、导出

### 2.4 列表展示字段

| 字段 | 说明 |
|------|------|
| 序号 | 行号 |
| 方案编号 | 关联的方案编号 |
| 项目编码 | 关联的项目编号 |
| 项目名称 | 项目名称（长文本展示） |
| 患者姓名 | 患者姓名 |
| 匹配状态 | 识别中（灰色）/ 匹配成功（绿色）/ 匹配失败（红色） |
| 匹配人 | 执行匹配的用户 |
| 操作时间 | 最近一次操作时间 |
| 开始匹配时间 | 匹配开始执行时间 |
| 结束匹配时间 | 匹配结束时间 |
| 操作 | 匹配详情、重新匹配 |

### 2.5 匹配详情页面

点击"匹配详情"进入，展示完整匹配运行信息，页面包含以下区块：

| 区块 | 内容 |
|------|------|
| 匹配运行信息 | AI模型、开始匹配时间、结束匹配时间、RAG用时、LLM用时、总用时 |
| 方案信息 | 方案编号、项目名称、入排标准版本（含上次更新时间） |
| 匹配度 | 入选标准匹配情况（通过数、待定义数）、排除标准匹配情况（通过数、待定义数） |
| AI建议 | AI给出的匹配建议（通过/筛败），附带建议说明 |
| 匹配详情列表 | 逐条展示每条入排标准的匹配结果，包含：序号、标准类型（入选标准/排除标准）、纳排内容、源数据、AI判断（RAG判定结果）、操作（病历溯源） |

### 2.6 重新匹配

1. 点击"重新匹配"按钮，前端调用后端匹配接口
2. 接口调用期间按钮显示加载状态，避免重复点击
3. 匹配完成后返回列表页，最新匹配记录自动更新

**输入**：
- 筛选查询条件（患者编号、匹配状态、匹配人、项目编码、时间范围等）
- 重新匹配操作触发后端接口调用

**输出**：
- 匹配日志（最新）列表（分页，支持每页10条/20条/50条切换）
- 匹配详情页面（完整匹配运行信息和逐条标准判定）
- 日志导出

**前置条件**：操作人具有匹配日志查看权限

**后置条件**：重新匹配后，最新匹配记录自动更新

**异常处理**：
- 查询数据量过大时限制单次导出行数
- 重新匹配接口调用失败时提示错误信息

## 五、数据从哪来

1. **匹配日志数据**：来自后端匹配服务，每次患者匹配时自动记录
2. **筛选表数据**：来自后端匹配服务，每次筛选表生成时自动记录
3. **筛选状态**：实时从后端接口获取，包括识别中、匹配成功、匹配失败

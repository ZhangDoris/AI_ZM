# HTML文件合并工具

## 📖 项目简介

这是一个用于将多个HTML文件智能合并为一个文件的Web工具。特别适用于需要将列表页和详情页合并到单个HTML文件的场景。

## 🎯 功能特性

### 核心功能
- ✅ **智能合并**：自动解析并合并两个HTML文件的结构
- ✅ **CSS合并**：合并所有样式到头部，去除重复样式
- ✅ **JavaScript合并**：合并所有脚本到底部，避免冲突
- ✅ **导航添加**：自动生成页面切换导航按钮
- ✅ **代码优化**：去除重复的meta标签和冗余代码

### 高级特性
- 🔍 **实时预览**：合并后可直接在预览区域查看效果
- 📊 **统计信息**：显示总行数、文件大小、合并耗时
- 📋 **详细日志**：记录合并过程的每一步操作
- 💾 **一键下载**：直接下载合并后的HTML文件

## 🚀 使用方法

### 步骤1：打开工具
在浏览器中打开 `html-merger/index.html` 文件

### 步骤2：选择文件
1. 点击"主文件（列表页）"选择第一个HTML文件
2. 点击"详情文件（详情页）"选择第二个HTML文件

### 步骤3：配置选项
根据需要勾选合并选项：
- ☑️ 合并所有CSS样式到头部
- ☑️ 合并所有JavaScript代码到底部
- ☑️ 添加页面导航切换按钮
- ☑️ 优化代码结构（去除重复标签）

### 步骤4：开始合并
点击"开始合并"按钮，等待处理完成

### 步骤5：预览和下载
- 在预览区域查看合并效果
- 点击"下载合并后的文件"保存结果

## 🔧 技术实现

### 核心技术
- **DOMParser API**：解析HTML字符串为DOM对象
- **Document Implementation**：创建新的HTML文档
- **Blob API**：生成可下载的文件
- **FileReader API**：读取本地文件内容

### 合并算法
1. 解析两个HTML文件为DOM树
2. 以第一个文档为基础结构
3. 智能合并第二个文档的样式和脚本
4. 去重检查（样式、脚本、meta标签）
5. 注入页面导航组件
6. 序列化为最终HTML字符串

## ⚠️ 注意事项

1. **文件编码**：确保源文件使用UTF-8编码
2. **外部资源**：合并后的文件可能仍依赖外部CSS/JS文件
3. **内联脚本**：某些内联脚本可能需要手动调整作用域
4. **ID冲突**：如果两个文件有相同的元素ID，需要手动处理

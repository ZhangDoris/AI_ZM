# 改进分析模块需求文档

版本：V1.0
日期：2026-06-15

---

## 一、这个模块是干啥的

简单说，就是帮用户看每次跑完筛选后，结果怎么样，哪里可以改得更好。

用户每次跑完项目，系统会给出一些建议，告诉用户哪些地方可以调整。用户看了建议，觉得有道理就改，改完再跑一次，看看结果有没有变好。

这个模块就是管这件事的：设置怎么跑、跑了之后看报告、看建议、查记录。

---

## 二、模块长什么样

左边有个导航栏，点"改进分析"会展开四个子页面：

1. **分析设置** - 配参数、手动跑一下
2. **分析报告** - 看每个项目的结果列表，点进去看详情
3. **改进意见管理** - 看系统给了哪些建议，哪些改了哪些没改
4. **执行日志** - 看每次跑的记录

## 三、每个页面具体要啥

### 3.1 分析设置页面（只给管理员）

**前置条件（重要）：**
只有满足以下条件的项目，才能进行分析设置和运行：
1. **项目已上架**：项目状态为"已上架/运行中"
2. **已有患者入组**：项目中至少有患者数据进入筛选流程
3. **有人工审核数据**：至少有一条患者记录完成了人工审核（CRC 提交、CRA 审核）
4. **有批注数据**：人工审核过程中留下了批注（审核意见、修正记录等）

**不满足上述条件的项目，系统不会进行分析对比。** 因为对比分析需要依赖人工审核结果作为"金标准"，来判断 AI 匹配的准确性，找出可以改进的地方。

**页面上有啥：**

1. 展示默认的输入字段，只是用于看，不能改
   - 哪些数据要喂给ai去分析
   - 必填的标红，可选的不标

2. 自动运行设置
   - 开关：想不想让它自己跑
   - 选星期几跑（周五到周日）
   - 几点钟跑（比如每天晚上10点）
   - 显示上次啥时候跑的，下次打算啥时候跑

3. 手动运行按钮
   - 有个下拉框，选哪个项目就跑哪个，选"全部"就跑全部
   - 点了按钮之后，按钮变灰显示"执行中..."
   - 跑完了弹个窗告诉你跑完了，一共跑了几个项目、多少条数据
   - **注意**：下拉框只会显示满足前置条件的项目，不满足条件的项目不会出现在列表中

**用户操作顺序：**
进页面 → 看字段 → 设自动运行（可选）→ 想马上看结果就点手动跑 → 跑完去报告页看

**为什么需要这些前置条件？**
分析模块的核心价值是"对比"——对比 AI 的判断和人工审核的结果，找出差异和改进空间。如果没有人工审核和批注数据，就无法知道 AI 判断得对不对，也就无法给出有价值的改进建议。

### 3.2 分析报告页面（只给管理员）

**页面上有啥：**

上面有一排搜索和排序：
- 搜项目号或项目名称
- 按病例数从多到少排，或从少到多排
- 按准确率从高到低排，或从低到高排

下面是个表格，每行一个项目，列表支持横向滚动，“项目编号”和“操作”两列固定不滚动，列包括：
- 项目编号（固定列）
- 方案编号
- 项目名称
- 病例数（这个项目有多少条数据）
- AI准确率（系统判断得准不准，百分数）
- 问题数（系统发现有问题的数量）
- 建议数（系统给了多少条改进建议）
- 已修改数（用户已根据建议实际修改的条数）
- 操作（“历史记录”按钮，点击弹出右侧浮窗查看该项目的历史记录分析）

点表格某一行，跳到详情页看这个项目的具体分析。点“历史记录”按钮，右侧弹出浮窗展示该项目的准确率对比、全部历史记录表格、提示词变更记录、准确率趋势图、指标追踪分析和图例说明。

---

### 3.3 报告详情页（只给管理员）

**页面上有啥：**

分两步走：

**第一步：核心指标和判断对比**
- AI准确率和AI判断错误率（两个卡片展示）
- AI判断 vs 人工判断 通过率对比
- 判断对比矩阵（一致通过、一致拒绝、误拒、漏拒）

**第二步：改进方案**
- 列出系统发现的问题，一个一个说
- 每个问题配上建议怎么改
- 每条建议显示修改状态标签：
  - <span style="color:green">已修改</span>：用户已实际修改提示词，附带修改日期
  - <span style="color:orange">未修改</span>：系统已发现但用户尚未修改
  - <span style="color:gray">待评估</span>：等待用户评估是否采纳
- 每条建议同时标记“已跑出 X 次”以及“第 X 次修改”，让用户清楚该建议的历史跟踪情况
- 支持编辑改进方案和提示词，切换预筛/入排开关
- 保存后用于下一次AI提示词更新

### 3.4 历史记录浮窗（从报告列表“历史记录”按钮弹出）（只给管理员）

**这是一个居中弹出的浮窗，带半透明遮罩。左侧为内容区，右侧为PRD说明栏。**

面板分六块：

**第一块：准确率对比分析**
- 两个卡片：AI准确率、AI判断错误率（错误率 = 100 - 准确率）
- 每个卡片显示上次→本次对比，带涨跌标识

**第二块：历史记录表格**
这个表格列了这个项目所有跑过的记录，从最新到最旧：

| 列名 | 说明 |
|------|------|
| 运行ID/时间 | 每次跑的编号和时间 |
| 提示词版本 | 用的是第几版规则 |
| 病例数 | 这次跑了多少条数据 |
| AI准确率 | 这次判断得准不准 |
| 提示词修改 | 系统建议改几条，实际改了几条。比如“发现3条，实际修改2条” |
| 状态 | 最新的那次标“当前”，其他的空白 |

准确率那一列，每次跑完都会跟上次比一下，用颜色区分：
- 90%以上绿色
- 80%到90%蓝色
- 80%以下橙色

**第三块：准确率趋势图**
- 折线图展示历史准确率变化趋势
- 标注最高点和最低点
- 底部显示最高/最低准确率摘要

**第四块：指标追踪分析**
这个表格看每个指标的情况：

| 列名 | 说明 |
|------|------|
| 指标名称 | 比如"他汀类药物判定"、"甲状腺功能"这些 |
| 运行次数 | 这个指标一共跑过多少次 |
| 人工修改状态 | 如果有人改过显示"已修改X次"，没改显示"-" |
| 追踪详情 | 两种情况：<br>1. "直接通过" - 没人改，系统自己跑出来的<br>2. "修改后重跑X次" - 有人改过，改完又跑了X次 |

**第五块：图例说明**
解释上面表格里各种图标和颜色的意思：
- "已修改"是啥意思
- "直接通过"是啥意思
- "修改后重跑"是啥意思

**面板怎么关：**
- 点右上角的"关闭"按钮
- 点面板外面的灰色遮罩
- 按键盘ESC键

### 3.5 改进意见管理页面（只给管理员）

**这是个监督视角的页面，看所有项目收到的建议。**

上面有搜索框：
- 搜建议内容
- 搜项目名
- 搜建议状态（已修改/保持现状）

下面是个表格，每行一条建议：

| 列名 | 说明 |
|------|------|
| 序号 | 1、2、3... |
| 建议内容 | 系统给的具体建议，比如"放宽他汀类判定" |
| 处理方式 | 已修改 / 保持现状 / 待评估 |
| 涉及项目 | 这条建议影响哪些项目 |
| 提出时间 | 什么时候给的建议 |
| 提出人 | 谁看的这条建议 |
| 同步状态 | 已同步到其他项目 / 未同步 |
| 操作 | 查看详情、编辑、同步 |

点"查看详情"会弹个框，显示这条建议的完整信息和分析过程。

点"编辑"可以改这条建议的处理方式和备注。

---

### 3.6 执行日志页面（只给管理员）

**记录每次跑的详细信息。**

上面有搜索框：
- 按时间范围查
- 按项目查
- 按类型查（手动跑的还是自动跑的）

下面是个表格：

| 列名 | 说明 |
|------|------|
| 执行时间 | 啥时候跑的 |
| 类型 | 手动 / 定时自动 |
| 项目数 | 这次跑了几个项目 |
| 病例总数 | 一共处理了多少条数据 |
| 耗时 | 跑了多久（比如3分42秒） |
| 状态 | 成功 / 失败 |

点某一行可以看这次运行的详细报告，包括每个项目的情况。

### 3.7 企业微信通知页面（只给医学经理）

**这是用户在企业微信里收到的消息通知页面，用于推送改进建议分析结果。**

**页面入口：**
用户在企业微信工作台点击"AI智能筛选系统"→ 收到改进建议分析通知消息

**页面上有啥：**

1. **顶部导航栏**
   - 显示"AI智能筛选系统"标题
   - 紫色渐变背景，与企业微信风格一致

2. **消息列表区域**
   - 按时间分组显示通知消息（最新在前）
   - 每条消息包含：
     - **消息标题**：项目名称 + "AI改进建议分析"（如"肺癌-9521-201 AI改进建议分析"）
     - **消息内容**：简要说明分析结果，如"基于您的审核结果对比了AI与人工分析，发现 **2 条** 可优化的改进建议。请查看并协助调整，让规则更加精准。"
     - **管理按钮**："管理改进建议"按钮，点击跳转到改进建议编辑页面（`modules/ai-improve.html`）

3. **改进建议弹窗（从管理按钮进入）**
   - 点击"管理改进建议"按钮后，弹出改进建议编辑窗口
   - 弹窗包含：
     - **进度指示**：显示当前第几条/总共几条（如"第 1 / 2 条"）
     - **优先级标签**：P0（红色）或 P1（橙色），标识建议的重要程度
     - **原因描述**：说明为什么提出这个建议（如"入选标准2:对他汀使用的判定过于严格"）
     - **改进方案**：可编辑的文本框，填写具体改进方案
     - **提示词**：可编辑的文本框，填写对应的提示词
     - **开关按钮**：
       - "预筛"开关：决定是否将此建议应用到预筛环节
       - "入排"开关：决定是否将此建议应用到入排标准
     - **操作按钮**：
       - 中间页："下一条"、"保存并下一条"
       - 最后一页："上一条"（非第一条时）、"保存"（未保存时）、"关闭"
   - 用户逐条查看、编辑、保存改进建议

**用户操作顺序：**
企业微信收到通知 → 点击消息 → 查看分析结果 → 点击"管理改进建议" → 逐条查看建议 → 编辑改进方案和提示词 → 切换预筛/入排开关 → 保存并关闭

**为什么需要这个通知页面：**
1. **及时触达**：用户不需要主动登录查看，企业微信主动推送分析结果
2. **便捷操作**：点击通知即可跳转到改进建议编辑页，无需登录系统
3. **逐条处理**：弹窗支持逐条查看和编辑建议，避免信息过载
4. **灵活应用**：通过预筛/入排开关，用户可以选择将建议应用到哪些环节

## 四、数据从哪来

### 4.1 项目数据
从系统里现有的项目列表拿，包括项目编号、方案编号、项目名称、病例数这些。

### 4.2 准确率数据
每次跑完分析，系统会根据判断结果和实际结果对比，算出一个准确率百分比。

### 4.3 建议数据
系统分析完之后，会给出一堆建议，告诉用户哪里可以改、怎么改。这些建议会存下来，用户可以决定改还是不改。

### 4.4 运行记录
每次跑分析都会留一条记录，包括时间、版本、用了多少数据、结果怎么样。

## 五、一些细节要求

### 5.1 页面跳转逻辑
- 侧边栏点"改进分析" → 进分析设置页
- 分析设置页点"查看报告" → 进报告列表页
- 报告列表页点某一行 → 替换当前页面跳转到报告详情页（ai-improve-report-detail.html）
- 报告列表页点“历史记录”按钮 → 居中弹出历史记录浮窗
- 报告详情页点“返回列表” → 通过浏览器后退回到报告列表页
- 历史记录浮窗点“关闭”或点遮罩 → 浮窗关闭

### 5.2 状态显示
- 正在跑的时候，按钮变灰，显示"执行中..."
- 跑完了，按钮恢复，状态变成"已完成"
- 跑失败了，状态变成"失败"，显示错误原因

### 5.3 颜色规范
- 准确率90%以上用绿色
- 准确率80%-90%用蓝色
- 准确率80%以下用橙色
- 错误/失败用红色
- 成功用绿色
- 警告/待处理用黄色

### 5.4 表格交互
- 鼠标移到表格行上，背景变浅灰（hover效果）
- 表头固定，内容太多可以横向滚动
- 搜索框输入后实时过滤表格内容

### 5.5 弹窗/面板
- 从右边滑出来的面板，要有半透明遮罩
- 点遮罩、点关闭按钮、按ESC都能关掉
- 弹窗里面内容太多可以滚动

## 七、怎么算做完了

以下情况都算通过：

1. 侧边栏能点"改进分析"，展开四个子页面
2. 分析设置页能选字段、设自动运行、手动跑一下
3. 报告列表页能搜、能排序、能看到所有项目，支持横向滚动，“项目编号”和“操作”两列固定
4. 点报告行能进详情页看具体分析
5. 报告列表页操作列“历史记录”按钮能弹出居中浮窗，包含准确率对比、历史记录表格、提示词变更、准确率趋势图、指标追踪分析和图例说明
6. 历史记录表格里“提示词修改”列显示“发现X条，实际修改Y条”
7. 指标追踪分析没有"是否已跑出"这一列了
8. 改进意见管理页能看建议、能搜索
9. 执行日志页能看每次运行记录、能按时间筛选
10. 所有弹窗和面板都能正常关掉
11. 颜色、字号、间距看起来舒服，不挤
12. 医学经理能在企业微信收到改进建议通知
13. 点击通知能跳转到改进建议编辑页
14. 弹窗能逐条展示改进建议，支持编辑和开关切换
15. 保存成功后能关闭弹窗
16. 报告详情页能看到准确率趋势折线图，支持选择时间范围
17. 趋势图能标出最高点和最低点，鼠标悬停显示详细信息
18. 报告详情页第二步的改进建议显示修改状态标签（已修改/未修改/待评估）
19. 已修改的建议显示修改日期，未修改的建议橙色标记突出显示
20. 每条建议标记“已跑出 X 次”及“第 X 次修改”，反映该建议的历史跟踪情况
